# EVT极值止盈 - 专业级设计方案

## 一、主流量化机构的EVT实现方式

### 1.1 文艺复兴(Renaissance Technologies)模式
- **滚动窗口**：1000-5000根K线
- **更新频率**：每100根K线更新一次参数
- **分布拟合**：使用GPD(广义帕累托分布)而非GEV
- **触发条件**：价格突破历史极值分位数时动态调整止盈

### 1.2 Two Sigma模式
- **多时间框架**：1m/5m/15m/1h同时计算
- **贝叶斯更新**：实时更新分布参数
- **自适应置信度**：根据胜率动态调整分位数(90%-99%)

### 1.3 桥水(Bridgewater)模式
- **尾部风险加权**：止盈位与VaR(风险价值)联动
- **宏观因子调整**：根据波动率 regime调整极值阈值

## 二、推荐方案：GPD-POT(峰值超过阈值)方法

### 为什么选择POT而非Block Maxima？
| 特性 | Block Maxima | POT(Peaks Over Threshold) |
|------|-------------|---------------------------|
| 数据利用 | 只取极大值，浪费数据 | 所有超过阈值的点都使用 |
| 样本量 | 需要大量数据 | 中等数据量即可 |
| 实时性 | 差 | 好 |
| 计算复杂度 | 低 | 中等 |

**结论**：POT更适合1分钟高频交易

## 三、系统架构设计

### 3.1 模块结构
```
V12系统
├── SignalGenerator (信号生成)
├── RiskManager (风控)
├── PositionManager (持仓管理)
└── EVTTakeProfitEngine (新增EVT止盈引擎)
    ├── GPDFitter (GPD分布拟合)
    ├── ThresholdSelector (阈值选择器)
    ├── DynamicTPAdjuster (动态止盈调整器)
    └── PerformanceTracker (绩效追踪器)
```

### 3.2 核心算法流程
```
每周期：
1. 收集收益率数据(滚动500根)
2. 选择阈值(95%分位数)
3. 拟合GPD分布
4. 计算目标止盈位 = 阈值 + GPD极值
5. 与现有ATR止盈对比，取更优者
6. 输出最终止盈位
```

### 4.2 EVT止盈的最佳位置
**建议插入位置**：第4步（分级止盈）之前

```python
# 修改后的止盈检查顺序：
1. 动态止损 (ATR倍数) - 必须最先
2. 盈利保护 (回撤50%) - 保护利润
```

- 在移动止盈之后：允许利润先奔跑
- 在固定ATR止盈之前：优先使用更智能的EVT
- 在ML反转之前：避免ML过早反转
