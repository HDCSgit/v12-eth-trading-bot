## 3. ML信号质量评估

### 3.1 置信度分布

从日志观察到的ML置信度分布：

| 置信度区间 | 出现次数 | 典型值 |
|-----------|---------|--------|
| 0.50-0.60 | 频繁 | 0.50 (观望) |
| 0.60-0.70 | 较少 | 0.66 |
| 0.70-0.80 | 中等 | 0.77 |
| 0.80-0.90 | 较少 | 0.86 |
| 0.90+ | 很少 | - |

### 3.2 置信度与胜率关系

**观察到的现象**:
- 高置信度(0.77+)信号开仓后，短期内价格波动较小
- 低置信度(0.50)时多为HOLD信号
- 当前市场处于"低波动休整"状态，ML难以给出强信号

## 4. 问题诊断

### 4.1 当前问题

| 问题 | 表现 | 根因分析 |
|-----|------|---------|
| **胜率低** | 10% (目标40%+) | 市场处于低波动状态，假突破多 |
| **频繁止损** | 8/10笔触发止损 | 止损设置可能过于敏感 |
| **持仓时间短** | 多数<10分钟 | 市场波动小，快速触及止损 |
| **ML信号弱** | 多数0.50置信度 | 震荡市特征不明显 |

### 4.2 ML模型局限

1. **训练数据不足**
   - 当前样本数: 169
   - 建议最小样本: 500+
   - 模型尚未充分学习

2. **特征可能过时**
   - 主要依赖: trend_short, macd_hist, momentum_5
   - 在1分钟级别，这些特征可能滞后

3. **阈值设置**
   - 未来3根K线(3分钟)预测
   - 0.15%阈值可能过于严格
