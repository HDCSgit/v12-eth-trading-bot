class MarketRegime(Enum):

    SIDEWAYS_UP   = "震荡上行"      # 做多为主

### 2.2 环境判断逻辑

```
判断流程:

3. 高波动/低波动检测 (ATR% + 成交量)
4. 趋势反转检测 (顶底背离)
5. 盘整检测 (布林带收窄 + ADX低)
6. 标准趋势检测 (ADX + 均线排列)
7. 震荡细分 (布林带 + 均线斜率)
```

## 三、入场策略详解

### 3.1 趋势市策略 (TRENDING_UP/DOWN)

#### 核心原则：**只顺势交易**

| 条件 | 顺势做多 | 顺势做空 |
|-----|---------|---------|
| 市场环境 | TRENDING_UP | TRENDING_DOWN |
| 均线排列 | MA10>MA20>MA55 | MA10<MA20<MA55 |
| ADX | > 25 | > 25 |
| ML置信度 | ≥ 0.56 | ≥ 0.56 |
| 逆势阈值 | N/A | N/A |

**ML逆势交易** (高置信度抓回调):
- 阈值: ≥ 0.75 (比普通顺势要求高)
- 用途: 捕捉趋势中的小幅回调/反弹
- 风险: 标记为"逆势(高置信度)"

**技术指标**:
- 只做顺势交易
- 条件: 均线排列 + 价格健康回调/反弹 + RSI健康范围
- **禁止逆势交易** (已修复)

### 3.2 震荡市策略 (SIDEWAYS)

#### 核心原则：**高抛低吸，顺势为主**

```python
# 普通震荡 (SIDEWAYS)
下轨做多: 价格<下轨×1.01 + RSI<40 + (RSI6<30 或 成交量>1.2倍)
上轨做空: 价格>上轨×0.99 + RSI>60 + (RSI6>70 或 成交量>1.2倍)

# 震荡上行 (SIDEWAYS_UP) - 只做多
下轨做多: 条件放宽 (RSI<45)
上轨做空: ❌ 禁止

# 震荡下行 (SIDEWAYS_DOWN) - 只做空
下轨做多: ❌ 禁止
上轨做空: 条件放宽 (RSI>55)
```

### 3.3 特殊行情策略

#### 插针暴跌 (BREAKDOWN)
- 触发: 3分钟跌 > 1.5% + 价格创20分钟新低 + 放量
- 策略: 快速抄底或追空
- 风控: 严格止损

#### 爆拉行情 (PUMP)
- 触发: 3分钟涨 > 1.0% + 接近20分钟高点 + 放量
- 策略: FOMO跟随追涨
- 风控: 移动止盈

#### 突破行情 (BREAKOUT)
- 触发: 突破20分钟高点 + ADX>35 + 放量 + 均线多头
- 策略: 追涨
- 确认: 必须放量

#### 盘整待破 (CONSOLIDATION)
- 触发: 布林带持续收窄 + ADX低
- 策略: **暂停交易** (当前禁用，因为之前0胜率)

## 五、风险管理策略

### 5.1 仓位计算

**公式**:
```
基础风险 = 余额 × MAX_RISK_PCT (默认3%)
风险金额 = 基础风险 × 置信度倍数
止损幅度 = ATR × 止损倍数 / 价格
仓位 = 风险金额 / 止损幅度

置信度倍数:
- 极高 (>0.8): ×3.0
- 高 (0.7-0.8): ×2.0
- 中 (0.6-0.7): ×1.5
- 低 (<0.6): ×1.0

限制:
- 最小: 30% 余额
- 最大: 80% 余额
```

### 5.2 止损止盈

| 趋势下跌 | 2.0 | 4.0~6.0 |
| 震荡市 | 1.5 | 中轨 |
| 突破 | 2.0 | 8.0 |

## 十、总结

### 策略优势

1. **多层风控**: ML+技术指标+市场环境+资金费率
2. **顺势为主**: 趋势市严格执行顺势原则
3. **ETH特化**: 针对ETH高波动优化
4. **自动学习**: ML模型定期重新训练

1. **震荡市误判**: 可能将趋势判断为震荡
2. **ML过拟合**: 时间特征权重过高
3. **逆势交易**: 需严格监控修复效果
4. **滑点风险**: 高波动时市价单成本
