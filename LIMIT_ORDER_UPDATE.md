# 限价单交易更新说明

## 更新日期
2026-03-27

## 核心改动

### 1. 交易机制升级
- **旧机制**: 100% 市价单 (Taker 0.05%)
- **新机制**: 限价单优先 (Maker 0.02%)，失败转市价单

### 2. 重试策略
```
第1次: 挂价偏离 0.05% (Maker最优)
   ↓ 等待0.8秒
第2次: 挂价偏离 0.02% (更接近市价)
   ↓ 等待0.8秒
失败: 转为市价单 (Taker 0.05%)
```

### 3. 订单类型标识
系统会记录每笔交易的订单类型：
- **MAKER** 💚: 限价单成交，手续费0.02% (节省60%)
- **HYBRID** 💛: 部分限价+部分市价
- **TAKER** ❤️: 市价单成交，手续费0.05%

## 配置参数 (config.py)

```python
"USE_LIMIT_ORDER": True,      # 启用限价单
"LIMIT_PRICE_OFFSET": 0.0005, # 价格偏离0.05%
"LIMIT_RETRY_MAX": 2,         # 最大重试2次
"LIMIT_WAIT_TIME": 0.8,       # 每次等待0.8秒
```

## 费用对比

| 场景 | 原市价单 | 新限价单 | 节省 |
|------|---------|---------|------|
| Maker成交 | 0.05% | 0.02% | **60%** |
| Taker成交 | 0.05% | 0.05% | 0% |
| 混合成交 | 0.05% | ~0.035% | ~30% |

## 预期效果

假设交易统计不变（21笔/天）：
- **原日手续费**: ~$0.80
- **预计新手續费**: ~$0.32~$0.50
- **日节省**: $0.30~$0.48
- **月节省**: $9~$14

## 注意事项

1. **成交率**: 限价单可能不如市价单立即成交，但系统会在1.6秒内完成2次尝试，确保及时成交

2. **价格偏移**: 
   - 0.05%偏离意味着买入价比市价低0.05%，卖出价比市价高0.05%
   - 在波动大的市场中更容易成交

3. **回退机制**: 两次限价失败自动转市价，不会无限等待

4. **日志标识**: 开仓/平仓日志会显示订单类型，方便追踪

## 数据库更新

已自动添加 `order_type` 字段到trades表：
```sql
ALTER TABLE trades ADD COLUMN order_type TEXT DEFAULT 'TAKER';
```

## 回滚方案

如需恢复市价单交易，修改config.py：
```python
"USE_LIMIT_ORDER": False
```
