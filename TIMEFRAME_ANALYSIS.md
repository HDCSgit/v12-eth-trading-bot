# EVT 时间尺度对比分析

## 当前：1分钟 K 线

### 特点
- 波动范围：0.05% - 0.3%（典型）
- EVT目标：0.6% - 1.0%（价格变动）
- 触发频率：高（可能每30-60分钟一次）

### 优势
1. **反应灵敏** - 快速捕捉短期波动
2. **机会多** - 一天可能有10-20次交易机会
3. **止损快** - 错误判断时快速离场

### 劣势
1. **目标太低** - 0.6%目标，费用0.5%，净利润仅0.1%
2. **噪声过大** - 1分钟波动随机性强，假信号多
3. **费用累积** - 频繁交易，手续费侵蚀利润
4. **滑点影响** - 快速波动时成交价格偏离

### 净利润估算
- 目标：0.8% × 5杠杆 = 4.0%
- 费用：0.1% × 2 × 5杠杆 = 1.0%
- 净利润：3.0%（看起来不错，但触发频繁）
- 每次净利润：$36 × 3% = $1.08
- 但一天10次交易 = $10.8费用累积

---

## 方案 A：5分钟 K 线

### 预估参数
- 波动范围：0.2% - 0.8%（典型）
- EVT目标：1.2% - 2.0%
- 触发频率：中（一天3-6次）

### 优势
1. **目标合理** - 1.5%目标，费用占比降至 0.5/7.5 = 6.7%
2. **噪声减少** - 过滤短期随机波动
3. **趋势更稳** - 5分钟趋势比1分钟更可靠
4. **费用降低** - 交易次数减少60-70%

### 劣势
1. **反应延迟** - 可能错过1分钟级别的快速波动
2. **持仓稍长** - 平均持仓时间从5分钟延长到15分钟
3. **需要更多数据** - 回测需要更多历史K线

### 净利润估算
- 目标：1.5% × 5杠杆 = 7.5%
- 费用：0.1% × 2 × 5杠杆 = 1.0%
- 净利润：6.5%
- 每次净利润：$36 × 6.5% = $2.34
- 一天4次交易 = $9.36总利润（比1分钟更稳定）

---

## 方案 B：15分钟 K 线

### 预估参数
- 波动范围：0.5% - 2.0%（典型）
- EVT目标：2.5% - 4.0%
- 触发频率：低（一天1-3次）

### 优势
1. **目标高** - 3%目标，净利润可达 15%-1% = 14%
2. **信号质量高** - 15分钟趋势更可靠，假信号少
3. **费用占比极低** - 1%费用 vs 15%盈利 = 6.7%
4. **适合上班/睡觉** - 不需要频繁监控

### 劣势
1. **反应慢** - 趋势反转时可能已经回撤较多
2. **持仓时间长** - 可能需要持仓30-60分钟
3. **机会少** - 一天可能只有1-2次交易
4. **资金利用率低** - 大部分时间空仓观望

### 净利润估算
- 目标：3% × 5杠杆 = 15%
- 费用：0.1% × 2 × 5杠杆 = 1.0%
- 净利润：14%
- 每次净利润：$36 × 14% = $5.04
- 一天2次交易 = $10.08总利润

## 综合对比

| 指标 | 1分钟 | 5分钟 | 15分钟 |
|------|-------|-------|--------|
| EVT目标 | 0.8% | 1.5% | 3.0% |
| 加杠杆盈利 | 4% | 7.5% | 15% |
| 费用占比 | 25% | 13% | 7% |
| **净利润** | **3%** | **6.5%** | **14%** |
| 交易频率 | 高(10次/天) | 中(4次/天) | 低(2次/天) |
| 信号质量 | 低 | 中 | 高 |
| 监控需求 | 极高 | 中 | 低 |
| 滑点影响 | 大 | 中 | 小 |

## 我的建议

### 推荐：5分钟 K 线 ✅

**理由：**
1. **风险收益比最优** - 净利润6.5%，费用占比13%
2. **交易频率适中** - 一天3-6次，不会错过主要波动
3. **信号质量提升** - 过滤噪声，减少假信号
4. ** still灵活** - 比15分钟更灵活，比1分钟更稳定
5. **适合你的资金量** - $36仓位，5分钟波动足够产生有意义盈利

### 不推荐：1分钟 ❌
- 目标太低，费用占比过高
- 频繁交易，心理压力大

### 备选：15分钟 ⭕
- 如果你工作/睡觉没时间盯盘
- 但可能错过日内多次机会

## 如果改为 5分钟，需要修改什么？

1. **数据获取** - 改为 get_klines(interval='5m')
2. **指标参数** - RSI、MA周期相应调整
3. **EVT窗口** - window_size 从500改为100（同样时长）
4. **阈值调整** - 止盈、止损阈值重新校准

这是一个较大的改动，需要重新回测验证。
