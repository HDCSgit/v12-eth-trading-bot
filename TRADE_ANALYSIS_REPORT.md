## 六、问题根因分析

### 6.1 核心问题：逆势交易过多

```
实际市场：震荡下行/插针下跌
系统行为：频繁抄底做多（16笔做多）
结果：逆势交易亏损
```

### 6.2 具体原因

1. **插针暴跌识别过于敏感**
   - 10笔插针交易，大部分不是真插针
   - 轻微下跌就触发插针模式

2. **震荡市细分（上行/下行）可能失效**
   - 5笔震荡市交易仅2胜
   - 方向判断可能错误

3. **逆势过滤不够严格**
   - 0.82阈值可能不够高
   - ML模型可能过度乐观

4. **凌晨时段交易过多**
   - 2-4点ETH流动性差，假信号多
   - 应该减少或禁止凌晨交易
