## 📈 核心指标解读

### 1. 特征重要性

**展示内容**: ML模型当前使用的Top 10特征及其权重

**如何解读**:
```
trend_short:  0.109  ← 最重要，短期趋势决定多空
macd_hist:    0.100  ← MACD柱状图，动量指标
momentum_5:   0.093  ← 5周期动量，快速响应
...
```

**优化建议**:
- 如果特征分布均匀 → 模型可能欠拟合
- 如果某个特征占主导(>30%) → 可能过拟合

---

### 2. 置信度分布

**展示内容**: 最近24小时ML信号的置信度分布

**关键观察点**:
- **均值**: 应该在0.6-0.8之间
- **高置信度占比(>0.8)**: 应该>20%
- **峰值位置**: 靠近0.5说明市场不明朗

**理想分布**:
```
低置信度(0.5-0.6):  ████ 20%
中置信度(0.6-0.8):  ██████████ 50%
高置信度(0.8-1.0):  ██████ 30%  ← 越多越好
```

---

### 3. 胜率趋势

**展示内容**: 最近20笔交易的滚动胜率

**关键线**:
- **红线(50%)**: 盈亏平衡
- **橙线(40%)**: 系统目标
- **绿柱**: 该笔交易盈利
- **红柱**: 该笔交易亏损

**健康状态**:
- 滚动胜率 > 40% → ✅ 正常
- 连续红柱 → ⚠️ 策略失效，需检查

---

### 4. PnL累积曲线

**展示内容**: 累积盈亏走势

**关键指标**:
- **总盈亏**: 曲线终点Y值
- **最大回撤**: 曲线从峰值回落的最大幅度
- **盈亏比**: 绿三角(盈利) vs 红三角(亏损) 比例

**理想形态**:
```
    ╱╲      ╱╲
   ╱  ╲    ╱  ╲____
  ╱    ╲__╱         ╲___  ← 向右上方倾斜
╱                         波动但总体向上
```

## 🔧 高级用法

### 自定义更新频率

**Matplotlib版**:
```python
# ml_visualizer.py 第39行
self.update_interval = 10  # 改为10秒
```

**Web版**:
```python
# ml_dashboard.py 第24行
interval=10*1000  # 改为10秒
```

### 调整显示时间范围

```python
# 修改SQL查询中的时间范围
WHERE timestamp >= datetime('now', '-48 hours')  # 48小时
```

### 添加自定义指标

在`ml_dashboard.py`中添加新的图表回调：

```python
@app.callback(
    Output('my-custom-chart', 'figure'),
    Input('interval-component', 'n_intervals')
)
def update_custom_chart(n):
    # 自定义图表逻辑
    ...
```

## 🐛 故障排除

### 问题1: 无法启动，提示缺少模块

**解决**:
```bash
pip install -r requirements.txt
```

### 问题2: Matplotlib窗口不显示

**可能原因**:
- 后端问题

**解决**:
```python
# ml_visualizer.py 第17行
matplotlib.use('TkAgg')  # 尝试改为 'Qt5Agg'
```

### 问题3: Web仪表盘端口被占用

**解决**:
```python
# ml_dashboard.py 第245行
app.run_server(port=8051)  # 换一个端口
```

### 问题4: 数据不更新

**检查**:
1. 数据库文件是否存在: `v12_optimized.db`
2. 数据库是否有新数据
3. 检查日志是否有错误

## 📊 实战案例

### 场景1: 发现胜率持续下降

**观察**:
- 胜率趋势图连续红柱
- PnL曲线向下

**行动**:
1. 检查特征重要性是否变化
2. 查看置信度分布是否左移
3. 考虑暂停交易，重新训练模型

### 场景2: 高置信度信号但亏损

**观察**:
- 置信度>0.8的 trades 亏损
- 特征重要性异常

**行动**:
1. 检查是否过拟合
2. 增加训练样本数
3. 调整ML阈值

### 场景3: 特征重要性变化

**观察**:
- trend_short 从0.1降到0.02
- volume_ratio 上升到0.15

**解读**:
- 市场环境变化，从趋势市转为量价驱动
- 需要调整策略权重

## 🎯 最佳实践

1. **同时运行两种可视化**
   - Matplotlib: 本地实时监控
   - Web: 深度分析和历史回溯

2. **设置警报阈值**
   - 胜率 < 30% → 警告
   - 回撤 > 10% → 停止交易
   - 连续5笔亏损 → 检查模型

3. **每日检查清单**
   - [ ] 胜率趋势是否正常
   - [ ] PnL曲线方向
   - [ ] 特征重要性稳定性
   - [ ] 置信度分布变化

4. **定期导出数据**
   ```bash
   sqlite3 v12_optimized.db ".dump" > backup_$(date +%Y%m%d).sql
   ```
