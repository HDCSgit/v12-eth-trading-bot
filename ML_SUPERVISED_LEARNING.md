# ML环境检测 - 可监督学习设计文档

## 1. 设计目标

实现**可监督的机器学习**：
- ✅ **事前记录**：记录ML预测和当时的市场状态
- ✅ **事后评估**：根据实际结果评估预测准确度
- ✅ **持续改进**：基于反馈自动调整模型参数
- ✅ **一键关停**：准确度低时自动建议关停

## 3. 核心组件

### 3.1 MLFeedbackSystem（反馈系统）

**文件**: `ml_feedback_system.py`

**功能**:
- 记录每次ML预测（环境类型、置信度、入场价）
- 交易完成后更新实际结果（出场价、盈亏、实际环境）
- 计算准确度统计
- 生成改进建议

**数据表**:
```sql
-- ml_predictions: 预测记录
-- ml_accuracy_stats: 每日准确度统计
-- ml_improvement_suggestions: 改进建议
```

### 3.2 主程序集成点

**集成点1**: 开仓时记录预测
```python
# execute_open中
if self.ml_feedback_enabled:
    from ml_feedback_system import record_ml_prediction
    self.current_prediction_id = record_ml_prediction(
        self.feedback_system,
        signal.ml_regime,
        signal.tech_regime,
        signal.final_regime,
        signal.ml_direction,
        signal.ml_confidence,
        entry_price
    )
```

**集成点2**: 平仓时更新结果
```python
# execute_close中
if self.ml_feedback_enabled and self.current_prediction_id:
    # 收集价格序列
    prices_5min = self.get_price_history(minutes=5)
    prices_15min = self.get_price_history(minutes=15)
    
    self.feedback_system.update_result(
        self.current_prediction_id,
        exit_price=price,
        actual_pnl=pnl_usdt,
        prices_5min=prices_5min,
        prices_15min=prices_15min
    )
```

### 3.3 准确度评估指标

| 指标 | 说明 | 用途 |
|------|------|------|
| **准确度** | 正确预测数 / 总预测数 | 评估整体性能 |
| **平均置信度** | 预测时的平均置信度 | 校准置信度阈值 |
| **盈亏对比** | 正确时 vs 错误时 | 评估实际价值 |
| **分环境准确度** | 各环境的准确度 | 识别薄弱环节 |
| **5/15分钟价格变化** | 短期验证 | 快速反馈 |

### 3.4 自动改进建议

系统会自动生成改进建议：

```
💡 模型改进建议:

  1. [threshold] 置信度高但准确度低，建议提高阈值
     当前: 0.75 → 建议: 0.80
     信心度: 80%

  2. [regime] STRONG_UP环境准确度低，建议禁用
     当前: enabled → 建议: disabled
     信心度: 70%
```

## 5. 配置参数

```python
# config.py - 可监督学习相关配置

# 反馈系统开关
"ML_FEEDBACK_ENABLED": True,  # 启用准确度跟踪
"ML_FEEDBACK_AUTO_SUGGEST": True,  # 自动生成改进建议

# 评估参数
"ML_EVALUATION_MIN_SAMPLES": 10,  # 最少样本数才评估
"ML_EVALUATION_WINDOW_DAYS": 7,   # 评估窗口（天）
"ML_ACCURACY_THRESHOLD": 0.50,    # 准确度警戒线（低于此值建议关停）

# 自动关停
"ML_AUTO_DISABLE_THRESHOLD": 0.40,  # 低于此准确度自动关停
```

## 6. 事后监督机制

### 6.1 预测时记录

```python
记录内容:
├── 时间戳
├── ML预测环境 (STRONG_UP/SIDEWAYS等)
├── 技术指标环境 (用于对比)
├── 最终使用环境 (可能被ML覆盖)
├── ML方向 (1/-1/0)
├── ML置信度 (0-1)
├── 入场价格
└── 状态: 未评估
```

### 6.2 事后评估（平仓后）

```python
评估内容:
├── 出场价格
├── 实际盈亏
├── 实际环境（事后判断）
├── 5分钟后价格变化
├── 15分钟后价格变化
├── 最大回撤
├── 最大盈利
├── 预测是否正确
└── 状态: 已评估
```

### 6.3 准确度计算

```python
# 简化规则
if 预测强趋势 and 实际盈利 > 0:
    正确 = True
elif 预测震荡 and |实际盈利| < 0.01:
    正确 = True
else:
    正确 = False
```

## 8. 总结

**可监督学习的价值**：

1. **透明度**：清楚知道ML判断的准确度
2. **可控性**：准确度低时可以一键关停
3. **持续改进**：基于反馈不断优化模型
4. **风险控制**：避免依赖不准确的ML判断

**使用建议**：

- 初期：启用反馈系统，但不依赖ML判断（覆盖开关关闭）
- 验证期：观察1-2周，积累数据
- 调整期：根据反馈调整参数
- 稳定期：准确度>60%后，正常启用
